Highlight #1 What is Version Control?

Version control is basically a respository that stores all of the project files. Different team members check the file for errors, make the necessary changes, and check them back in. Checking files back in is known as commiting the files.Version control is a fix to all file sharing problems.

Highlight #2 What is a veresionaing control system?

A version control system or VCS is a system that automatically notes and records many different functions. First it receords who made changes to what files, known as revision history. Second, It records when the files were changed. Third, it records what changes were actually made like what is new or different.Lastly, the system can merge changes to the same file.There are many benefits to using a VCS as well. These benefits include not allowing files to be overwritten, and a common repository.

Highlight #5 Remote and Local Repositories

A remote repository is a repsitory that resides on a remote server and can be accessed by all of the team members the files are shared with. On the other hand, a local repository is a repository that resides on an individuals server of an individual user. Local repositories can be created by either creating a new repository or cloning an existing one.
